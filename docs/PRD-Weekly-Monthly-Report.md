# PRD: 주간/월간 리포팅 시스템 v2.0

## 1. 개요

### 1.1 배경
현재 Polarad 대시보드는 실시간 데이터 조회와 기간별 필터링 기능을 제공하지만, **정기 리포트 생성 및 AI 기반 인사이트 분석** 기능이 없음. 클라이언트에게 주간/월간 단위로 **데이터 분석 전문가 수준의 체계적인 성과 리포트**를 제공하고, 관리자 의견을 추가할 수 있는 시스템 필요.

### 1.2 목표
- **주간 리포트**: 매주 월요일, 지난 주(월~일) 성과 분석
- **월간 리포트**: 매월 1일, 지난 달 전체 성과 분석
- **AI 인사이트**: Gemini 3 Pro Preview를 활용한 데이터 기반 분석 및 제안
- **관리자 의견**: 리포트에 수동으로 코멘트 추가 기능
- **전문가 수준 시각화**: 구조화된 차트, 데이터 스토리텔링, 맥락 분석

### 1.3 범위
| 포함 | 제외 |
|------|------|
| 주간/월간 리포트 생성 | 실시간 알림 시스템 개편 |
| AI 분석 인사이트 | 새로운 데이터 소스 추가 |
| 전문가 수준 데이터 시각화 | 클라이언트 직접 수정 기능 |
| 관리자 코멘트 기능 | PDF 다운로드 (Phase 2) |
| 리포트 조회 페이지 | 이메일 발송 (Phase 2) |
| 텔레그램 리포트 발송 | |

## 2. 데이터 시각화 설계 원칙

### 2.1 차트 선택 가이드라인

| 분석 목적 | 권장 차트 | 사용 예시 |
|----------|----------|----------|
| **시간 추이** | Line Chart | 일별/주별 노출수, 클릭수 변화 |
| **카테고리 비교** | Bar Chart (수평/수직) | 채널별 성과, 키워드별 CPC |
| **구성 비율** | Donut/Pie Chart | 채널별 지출 비중, 플랫폼 분포 |
| **목표 달성률** | Progress Bar / Gauge | KPI 달성률, 예산 소진율 |
| **상관관계** | Scatter Plot | CTR vs CPC, 노출 vs 클릭 |
| **변화율 강조** | Delta Indicator (▲▼) | 전 기간 대비 증감 |
| **순위/분포** | Horizontal Bar | 키워드 TOP 10, 광고 성과 순위 |
| **퍼널 분석** | Funnel Chart | 노출 → 클릭 → 전환 흐름 |

### 2.2 대시보드 설계 원칙

```
┌─────────────────────────────────────────────────────────────┐
```

## 4. 주간 vs 월간 리포트 차이점

| 구분 | 주간 리포트 | 월간 리포트 |
|------|------------|------------|
| **기간** | 지난 주 (월~일) | 지난 달 (1일~말일) |
| **비교 기준** | 전주 대비 | 전월 대비 |
| **상세도** | 일별 추이 중심 | 주별 추이 + 월간 패턴 |
| **차트** | 7일 라인차트 | 4~5주 라인차트 + 요일별 히트맵 |
| **AI 분석** | 단기 전술적 제안 | 중장기 전략적 제안 |
| **주요 섹션** | KPI + 채널비교 + 일별추이 | KPI + 채널비교 + 주별추이 + 요일패턴 + 트렌드 |

### 5.1 신규 테이블

```sql
CREATE TABLE polarad_reports (

  -- 기간
  period_start DATE NOT NULL,

  period_label VARCHAR(50), -- "2025년 12월 1주차", "2025년 11월"

  -- KPI 요약 (JSON)
  kpi_summary JSONB NOT NULL,

    "day_of_week_pattern": {...},

  -- AI 분석 결과 (JSON)

  -- 관리자 의견
  admin_comment TEXT,
  admin_comment_updated_at TIMESTAMP,
```

## 6. API 설계

### 6.1 리포트 생성
```
POST /api/reports/generate
Headers: x-admin-key: {admin_key}
Body: {
  "clientId": "uuid",
  "reportType": "weekly" | "monthly",
  "periodStart": "2025-11-01",  // optional
  "periodEnd": "2025-11-30"     // optional
}
Response: {
  "success": true,
  "report": { ... }
}
```

### 6.2 리포트 조회
```
GET /api/reports?clientSlug={slug}&type={weekly|monthly}&limit=10
GET /api/reports/{reportId}
```

### 6.3 관리자 의견 수정
```
PATCH /api/reports/{reportId}/comment
```

### 6.4 AI 인사이트 재생성
```
POST /api/reports/{reportId}/regenerate-insights
```

### 6.5 리포트 발행
```
POST /api/reports/{reportId}/publish
Body: { "sendTelegram": true }
```
